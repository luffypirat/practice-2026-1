# Телеграмм-бот для отслеживания задач
Проект представляет собой Телеграмм-бота для управления списком задач, разработанного на языке Python с использованием асинхронного фреймворка Aiogram и асинхронной базы данных SQLite. Бот позволяет пользователям создавать, просматривать, выполнять и удалять задачи через удобный интерфейс Телеграмм-команд и кнопок.

## Основная цель проекта
Предоставить простой и удобный инструмент для персонального управления задачами прямо в мессенджере Телеграмм без необходимости использования отдельных приложений.

---

# Основной функционал проекта

- Добавление новых задач
- Просмотр списка всех задач пользователя
- Отметка задач как выполненных
- Удаление одной задачи
- Удаление всех выполненных задач
- Работа через команды и интерактивные кнопки
- Поддержка состояний пользователя (FSM)
- Хранение задач в SQLite базе данных

---

# Используемые технологии

- Python
- Aiogram — асинхронный фреймворк для Телеграмм-ботов
- SQLite — локальная база данных
- FSM (Finite State Machine) — для обработки пользовательских состояний
- AsyncIO — для асинхронного выполнения операций

# Архитектура и логика работы проекта

Проект построен на принципе асинхронного Телеграмм-приложения, где пользователь взаимодействует с ботом через команды, кнопки и сообщения, а все данные о задачах хранятся в базе данных SQLite. После запуска бот соединяется с Телеграмм API и постоянно ожидает новых сообщений от пользователей.

Когда пользователь отправляет команду или нажимает кнопку, бот определяет тип действия и запускает соответствующий обработчик. Например, при добавлении задачи бот сначала переводит пользователя в специальное состояние ожидания ввода задачи. Такой подход реализован с использованием FSM (Finite State Machine) и позволяет боту понимать контекст диалога и правильно обрабатывать последовательность действий пользователя.

После получения текста задачи бот сохраняет её в базе данных вместе с идентификатором пользователя Телеграмм. Это позволяет каждому пользователю работать только со своим списком задач. Все операции с данными выполняются асинхронно, что позволяет быстро обрабатывать запросы и не блокировать выполнение других операций.

При просмотре списка задач бот получает данные из базы данных, формирует текстовый список и отображает состояние каждой задачи с помощью специальных индикаторов:
- ✅ — задача выполнена
- ❌ — задача не выполнена

Для более удобного взаимодействия используются inline-кнопки. При нажатии на такую кнопку Телеграмм отправляет callback-запрос, который позволяет мгновенно изменить статус задачи без ввода дополнительных команд.

Логика удаления задач построена аналогичным образом: бот получает номер задачи, проверяет её существование и удаляет запись из базы данных. Также реализована возможность массового удаления всех выполненных задач пользователя.

Внутри проекта используется модульный подход, при котором логика приложения, работа с базой данных и элементы интерфейса разделены между собой. Это делает код более понятным, облегчает поддержку проекта и упрощает добавление нового функционала.

В основе проекта лежит асинхронная архитектура Python с использованием AsyncIO, что позволяет эффективно обрабатывать большое количество запросов и делает приложение более производительным по сравнению с синхронными решениями.
